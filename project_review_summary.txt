# Project Review Summary: MARCO Framework
Date: March 29, 2026
Project Title: MARCO (Multimodal Agentic RAG for DevOps–Microservice Architecture Concept Alignment)

---

## 1. Project Overview & Objective
MARCO is an advanced research framework that extends traditional RAG (Retrieval-Augmented Generation) systems for DevOps and Microservices. It addresses the limitations of static, linear RAG by introducing:
- **Agentic Reasoning:** A self-correcting multi-agent workflow using LangGraph.
- **Multimodal Inputs:** Support for architectural diagrams, CI/CD YAML, and Infrastructure-as-Code (Terraform).
- **Adaptive Knowledge:** Automated ingestion of the latest research papers from arXiv.

---

## 2. Key Accomplishments (What We Have Done)

### A. Core Architecture (Agentic Loop)
- **Implemented a 4-Agent Workflow** using LangGraph:
    - **Planner:** Decomposes complex DevOps queries into sub-questions.
    - **Retriever:** Hybrid search using FAISS (dense) and BM25 (sparse) with a Cross-Encoder reranker.
    - **Critic:** Evaluates retrieval quality and triggers self-correction loops if relevance is low.
    - **Generator:** Synthesizes final recommendations using Gemini 2.0 Flash, grounded in retrieved evidence.
- **Self-Correction Logic:** Enabled the system to retry retrieval up to 2 times, significantly reducing failed queries.

### B. Multimodal Processing Engine
- **Vision Processor:** Integrated Gemini Flash Vision to extract DevOps/MSA concepts from architecture diagrams and dashboard screenshots.
- **Artifact Parsers:** 
    - **CI/CD Parser:** Extracts stages and tools from GitHub Actions/GitLab YAML files.
    - **IaC Parser:** Maps Terraform resources (AWS/K8s) to architectural patterns.
- **Concept Normalization:** All extracted data is mapped to a standardized DevOps-MSA Ontology (v1.0).

### C. Knowledge Base & Ingestion
- **Automated Ingestion Pipeline:** Built a daemon that pulls new papers from arXiv, extracts concepts using LLMs (Groq Llama-3), and updates the vector index incrementally.
- **Hybrid Vector Store:** Combined FAISS for high-speed similarity search with ChromaDB for persistent metadata management.

### D. User Interface
- **Streamlit Application:** Developed a functional UI (`ui/app.py`) allowing users to:
    - Input text-based queries.
    - Upload architecture artifacts (PDF/PNG/YAML/TF).
    - View the step-by-step reasoning trace of the agents.

---

## 3. Evaluation & Results

### Quantitative Performance
- **Reproduced Baseline:** Confirmed the base paper's P@5 (~0.82 with human grading).
- **Hallucination Control:** Achieved a **0.0% Hallucination Rate** across all trials, thanks to the Critic agent's verification.
- **Ranking Improvement:** The multimodal system (nDCG@5 = 0.3119) outperformed the text-only version (nDCG@5 = 0.2392), proving that artifact context improves recommendation quality.

### Ablation Study Findings
- **Impact of Critic Agent:** Removing the Critic led to a **50% drop in Precision@5**, validating the need for agentic self-correction.
- **Impact of Multimodal Data:** Artifact-grounded queries showed significantly higher concept recall than text-only queries.
